Imagens Dinâmicas

É importante termos a mesma imagem de tamanhos diferentes pois os usuários podem acessar nosso site por dispositivos diferentes e uma imagem muito pesada pode deixar o site mais lento por conta de uma imagem pesada, devemos tratar a imagem para que isso não ocorra, para isso, nesse exemplo criamos 3 imagens de tamanhos diferentes, a primeira é a de tamanho 1000 x 1000 px, que aparecerá para usuários que estão acessando o nosso site por um computador, outra imagem de tamanho 700 x 700 px, que será mostrada para usuários que acessarem nosso site através de tablets, e uma última de tamanhos 300 x 300 px, que irá ser mostrada para usuários que acessarem nosso site através do celular.

Podemos criar inúmeros tamanhos de imagens para adaptarmos.

Tag <Picture>

A tag <picture> deve abrigar a tag <img> que queremos que seja mostrada inicialmente. Ex:

<picture>
<img src="imagem-1050px.png" alt="imagem 1">
</picture>

No entanto se só escrevermos apenas isso, apenas a imagem que inserimos será mostrada, independente do tamanho dela, então devemos inserir antes da tag de imagem a tag <source> e inserir nessa tag o atributo "media" podendo definir uma tamanho máximo ou mínimo (max-width/min-width) que o navegador deve ter para determinada imagem permanecer sendo mostrada para o usuário, o valor inserido pode ser em pixels, devemos sempre colocar um valor superior ao da imagem para que haja uma folga (Ex: para imagens de 1000px max-width: 1050px), só lidaremos com max-width; também o atributo "srcset" onde inserimos o link para a imagem que desejamos que seja mostrada para o usuário caso o tamanho máximo que o navegador deva ter para mostrar a imagem inicial ao usuário seja passado; e por fim o atributo "type" para indicar o tipo da imagem o valor deve ser "image/exetenção do arquivo" (Ex: type="image/jpg"). obs: devemos colocar uma tag source para cada imagem que deve substituir outra sempre que o tamanho máximo exigido não seja alcançado, a ordem deve ser sempre a seguinte: se você colocou primeira a menor imagem então o primeiro source a ser escrito deve ser o para substituir a segunda maior imagem pela maior imagem, depois para substituir a terceira maior imagem para a segunda maior, e assim por diante até chegar a hora de aparecer a menor imagem (onde o tamanho do navegador será menor que o max-width necessário para exibir a segunda menor imagem); se a imagem que você inseriu foi a maior então o primeiro source deve ser para a menor imagem, depois para a segunda menor, e assim por diante até chegar a hora de aparecer a maior imagem (onde o tamanho do navegador será equivalente ao max-width necessário para exibir a maior imagem).

Ex:
<picture>
<source media="(max-width: 750px)" srcset="../imagem/imagem-300px.png" type="image/">
<source media="(max-width: 1000px)" srcset="../imagen/imagem-700px.png" type="image/">
<img src="imagem-1050px.png" alt="imagem 1">
</picture>

Testando a funcionalidade da Tag <Picture>

Tente abrir esse site em vários dispositivos diferentes ou simplismente aumente e diminua o tamanho do seu navegador.

Note que conforme diminuimos o tamanho do navegador a imagem vai sendo substituída conforme o tamanho limite exigido para a imagem permanecer sendo exibida vai sendo passado.

Usar esse sistema de imagem flexível ajuda muito os usuários a navegarem pelo nosso site.

A ordem das imagens deve estar certo, no exemplo acima, o primeiro source sempre será a menor imagem.

Inserção de Áudios na página Web

Para inserirmos um áudio na nossa webpage precisamos usar a tag <audio></audio>, essa tag funciona semelhantemente à tag <picture>, pois assim como ela, você precisa colocar uma ou mais tags <source> para inserir um áudio (normalmente usamos mais de um source para inserirmos o mesmo áudio em outros formatos, para caso o usuário não consiga ouvir o áudio no formato principal, porém diferente da piture, o primeiro source será o primeiro a ser carregado e somente se o primeiro áudio falhe os seguintes serão carregador também). Para inserirmosum audio escrevemos o seguinte código: 


            <audio preload="metadata" controls autoplay loop>
                <source src="../caminho/para-o-/áudio/-baixado.mjpeg" type="audio/mjpeg">
                <source src="../caminho/para-o-/áudio/-baixado.mp3" type="audio/mjpeg">
                <source src="../caminho/para-o-/áudio/-baixado.ogg" type="audio/mjpeg">
            </audio>

Vamos explicar a respeito de alguns atributos e valores inseridos na tag <audio>a seguir:

  • preload= esse atributo indica se o áudio será pré-carregado ou não e aceita três valores:
    1. metadata= vai carregar apenas as informações sobre o arquivo (tamanho, tempo, informações de direitos, etc)
    2. none= não vai carregar nada até que o usuário clique no botão de play ou um script inicie a reprodução
    3. auto=valor padrão, carrega o arquivo inteiro assim que a página for carregada, mesmo que o usuário nunca aperte play
  • controls= vai apresentar o player na tela. Caso não seja colocado na tag <audio>, o controle será transparente e o usuário não poderá interagir com ele
  • autoplay= quando inserido, vai indicar a reprodução do áudio assim que a página for carregada
  • loop= faz com que o áudio seja repetido eternamente assim que terminar a sua reprodução

Inserção de vídeos em sua página web

Para inserir vídeos em nosso site podemos utilizar a nova tag <video> da HTML5, caso o arquivo esteja hospedado em nosso próprio servidor (em caso de treino, no seu prórpio pc)

Veja abaixo um exemplo da tag sendo usada:


        <video width="600" poster="thumb.jpg" controls autoplay>
            <source src="meu-video.webm" type="video/webm">
            <source src="meu-video.mp4" type="video/mp4">
            <source src="meu-video.ogg" type="video/ogg">
        </video>

Abaixo estã algumas explicações sobre os atributos inseridos na tag <video>

  • width= esse atributo indica a largura que o video vai ter na tela (no exemplo acime o largura é 600px)
  • poster= configura uma imagem que vai aparecer como capa, enquanto o visitante não aperta o play para reproduzir o vídeo
  • controls= vai configurar se os controler do vídeo vão aparecer na parte inferior da mídia. Por padrão os controles não aparecerão, mas basta colocar a palavra "controls" na tag <video>
  • autoplay= diz ao navegador para o vídeo começar a tocar automaticamente, assim que a página for carregada.

Atualmente os dois formatos de vídeo mais suportados pelos navegadores atuais são o MPEG e o WEBM.

Abrigar um vídeo no próprio servidor pode causar problemas como o alto consumo de banda, site lento e incompatibilidades com alguns navegadores por conta dos codecs. Para isso não acontecer, podemos hospedar o nosso vídeo no youtube, que possui servidores muito rápidos, porém essa pode não ser uma boa escolha caso você queira limitar quem vai ter acesso a determinado vídeo (como por exemplo, uma aula online que queremos que somente os alunos matrículados tenham acesso), pois a ideia do Google é deixar todos esses vídeos públicos e disponíveis. Caso esse seja o caso, então podemos hospedar nosso vídeo no Vimeo que é uma plataforma que possibilita limitar quem vai poder ver o vídeo, o que é especialmente vantajoso no caso do exemplo citado acima. No entanto, o Vimeo possui uma taxa anual e seus algoritmos não são tão eficientes quanto os do YouTube (isso faz com que alguns vídeos apresentem leves travamentos às vezes).

Como imcorporar um vídeo do YouTube em HTML

Para incorporar um vídeo do YouTube em HTML basta, abrir o vídeo no YouTube, clicar na opção compartilhar, e vai aparecer um opção chamada "incoporar" que possui acima da palavra os símbolos "<>". O código HTML personalizado vai aparecer em uma nova janela de texto, incluindo um botão que permite COPIAR o código com a tag do <iframe> que vai aparecer diretamente na sua página. Após copiar o código, basta colar a tag no local desejado no seu código em HTML.
